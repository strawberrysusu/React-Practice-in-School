## 1. 이벤트 핸들러 작성 방식 2가지

### (1) 인라인 핸들러 - 이벤트 속성 안에 함수 직접 작성
```tsx
<button onClick={() => alert("클릭!")}>클릭</button>
```
- 간단하고 직관적
- 매개변수 전달이 자유로움

### (2) 함수 참조 - 함수를 밖에 정의하고 이름만 전달
```tsx
const clickHandler = () => {
  alert("클릭!");
};

<button onClick={clickHandler}>클릭</button>
//               ^^^^^^^^^^^^  괄호 없이 함수 이름만!
```

### 함수 참조 시 괄호 붙이면 안 되는 이유! (시험 주의)
```tsx
// 잘못된 코드 - 괄호 붙이면 즉시 실행됨!
<button onClick={clickHandler()}>클릭</button>
// → 버튼 클릭 안 해도 컴포넌트가 렌더링될 때 함수가 바로 실행됨

// 올바른 코드 - 괄호 없이 함수 참조만
<button onClick={clickHandler}>클릭</button>
// → 버튼 클릭할 때만 실행됨
```

**왜?**
- `clickHandler()` = 함수를 **지금 당장 실행**하고 그 결과(return 값)를 onClick에 넣음
- `clickHandler` = 함수 **자체를 전달** → 클릭 시점에 React가 호출

### 매개변수를 전달해야 할 때
```tsx
// 함수 참조로는 매개변수 전달 불가 → 인라인 핸들러 사용!
<button onClick={() => clickHandler("hello")}>클릭</button>
//              ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^ 화살표 함수로 감쌈
```

## 2. 이벤트 객체

### 합성 이벤트 (SyntheticEvent)
- React는 브라우저의 기본 DOM 이벤트를 **감싸서 만든 자체 이벤트 객체** 사용
- 왜? → 브라우저마다 이벤트 동작이 조금씩 달라서 **통일시키기 위해**
- 이를 "크로스 브라우저 호환성(cross-browser compatibility)"이라 함
- 기본 DOM 이벤트와 거의 동일하게 사용 가능

### 인라인 핸들러에서 이벤트 객체 받기
```tsx
<button onClick={(e) => console.log(e)}>클릭</button>
// e = 합성 이벤트 객체 (SyntheticEvent)
```

### 함수 참조에서 이벤트 객체 받기
```tsx
// React가 자동으로 첫 번째 매개변수로 이벤트 객체를 전달!
const clickHandler = (e: React.MouseEvent<HTMLButtonElement>) => {
  console.log(e);
};

<button onClick={clickHandler}>클릭</button>
// clickHandler(e) 라고 안 써도 React가 알아서 e를 넘겨줌
```

### 타입을 모르겠을 때? → 타입 추론 활용
```tsx
// 1단계: 인라인으로 먼저 써보기
<button onClick={(e) => console.log(e)}>클릭</button>

// 2단계: e 위에 마우스 올리면 VSCode가 타입을 보여줌
// → React.MouseEvent<HTMLButtonElement, MouseEvent>

// 3단계: 그 타입을 함수 참조에 복사
const clickHandler = (e: React.MouseEvent<HTMLButtonElement>) => { ... };
```

### 이벤트 객체 + 추가 매개변수 동시에 전달
```tsx
// 이벤트 객체 + 사용자 값을 함께 보내려면 → 인라인 핸들러 필수
const clickHandler = (e: React.MouseEvent<HTMLButtonElement>, msg: string) => {
  console.log(e.target);
  console.log(msg);
};

<button onClick={(e) => clickHandler(e, "Hello")}>클릭</button>
```

## 3. 이벤트 전파 (Event Propagation) - 시험 가능!

이벤트가 발생하면 해당 요소뿐 아니라 **부모/자식 요소에도 전달**될 수 있음.

### 캡처링 (Capturing) - 부모 → 자식
```
이벤트 발생!
  div (부모) ← 여기서 먼저 잡음
    ↓
  button (자식) ← 그 다음 도착
```

```tsx
// 캡처링을 사용하려면 이벤트 속성에 Capture를 붙임
<div onClickCapture={() => console.log("Parent (캡처링)")}>
  <button onClick={() => console.log("Child")}>
    클릭
  </button>
</div>

// 결과: Parent (캡처링) → Child
```

### 버블링 (Bubbling) - 자식 → 부모 (기본 동작!)
```
이벤트 발생!
  button (자식) ← 여기서 먼저 실행
    ↑
  div (부모) ← 그 다음 전파됨
```

```tsx
// 버블링은 기본 동작이라 특별한 설정 없이 일어남
<div onClick={() => console.log("Parent (버블링)")}>
  <button onClick={() => console.log("Child")}>
    클릭
  </button>
</div>

// 결과: Child → Parent (버블링)
```

### 정리: 캡처링 vs 버블링

| | 캡처링 | 버블링 |
|---|---|---|
| **방향** | 부모 → 자식 (위에서 아래로) | 자식 → 부모 (아래에서 위로) |
| **기본 동작?** | 아님 (명시적으로 설정해야) | **기본 동작** |
| **이벤트 속성** | `onClickCapture` | `onClick` |
| **실행 순서** | 캡처링이 먼저, 버블링이 나중 |

### stopPropagation() - 이벤트 전파 막기

```tsx
<div onClick={() => console.log("Parent")}>
  <button onClick={(e) => {
    e.stopPropagation();  // 이벤트가 부모로 전파되는 걸 막음!
    console.log("Child");
  }}>
    클릭
  </button>
</div>

// stopPropagation() 없으면: Child → Parent
// stopPropagation() 있으면: Child (Parent에 안 감!)
```

### preventDefault() vs stopPropagation() 차이!

| | preventDefault() | stopPropagation() |
|---|---|---|
| **무엇을 막나?** | 브라우저의 **기본 동작** | 이벤트의 **전파 (버블링/캡처링)** |
| **예시** | form 제출 시 페이지 새로고침 막기 | 자식 클릭 시 부모에 전파 안 되게 |
| **사용 상황** | form, a 태그 등의 기본 동작 차단 | 중첩된 요소에서 이벤트 분리 |

```tsx
// preventDefault: form 기본 동작(새로고침) 막기
<form onSubmit={(e) => {
  e.preventDefault();  // 페이지 새로고침 방지
  // 폼 처리 로직
}}>

// stopPropagation: 부모에 이벤트 전파 막기
<button onClick={(e) => {
  e.stopPropagation();  // 부모의 onClick 실행 안 됨
  // 버튼만의 로직
}}>
```

## 합격 기준

- "버블링이 뭐냐?" → "이벤트가 자식에서 부모로 전파되는 기본 동작" OK
- "캡처링은?" → "부모에서 자식으로, onClickCapture로 설정" OK
- "stopPropagation vs preventDefault 차이?" → "전파 차단 vs 기본 동작 차단" OK
- "함수 참조에서 괄호 붙이면 왜 안 되나?" → "즉시 실행돼서 렌더링 시점에 함수가 호출됨" OK
