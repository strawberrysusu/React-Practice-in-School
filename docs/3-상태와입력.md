# 3단계: 상태(State)와 입력

> React의 핵심! 상태를 바꾸면 화면이 바뀐다.

---

## 반드시 알아야 하는 것

### 1. useState 기본

```tsx
import { useState } from "react";

function Counter() {
  const [count, setCount] = useState(0);
  //     ↑현재값   ↑바꾸는함수         ↑초기값

  return (
    <div>
      <p>카운트: {count}</p>
      <button onClick={() => setCount(count + 1)}>+1</button>
    </div>
  );
}
```

**핵심 원리:**
```
setCount 호출 → React가 상태 변경 감지 → 컴포넌트 다시 실행 → 화면 업데이트
```
직접 DOM을 건드리는 게 아니라, **상태만 바꾸면 화면은 자동으로 따라온다!**

### 2. 제네릭으로 타입 지정

```tsx
const [name, setName] = useState<string>("");        // 문자열
const [count, setCount] = useState<number>(0);        // 숫자
const [isOpen, setIsOpen] = useState<boolean>(false);  // 참/거짓
const [items, setItems] = useState<string[]>([]);      // 문자열 배열

// 초기값으로 추론 가능하면 생략 OK
const [count, setCount] = useState(0); // 자동으로 number
```

### 3. 이벤트 처리 + 타입

```tsx
// 클릭 이벤트
<button onClick={() => setCount(count + 1)}>클릭</button>

// 입력 이벤트 (타입이 시험에 나옴!)
const handleChange = (e: React.ChangeEvent<HTMLInputElement>) => {
  setName(e.target.value);  // e.target.value = 입력된 문자열
};

// 폼 제출 이벤트
const handleSubmit = (e: React.FormEvent<HTMLFormElement>) => {
  e.preventDefault();  // 페이지 새로고침 방지!
  // 폼 처리 로직
};
```

**e.preventDefault() 왜 쓰나?**
> form은 기본적으로 제출하면 페이지가 새로고침됨.
> React는 SPA(싱글페이지앱)라서 새로고침하면 state가 전부 날아감.
> 그래서 기본 동작을 막아야 함!

### 4. Controlled Input (시험 필수!)

```tsx
const [name, setName] = useState("");

<input
  value={name}                              // state가 값을 결정
  onChange={(e) => setName(e.target.value)}  // 입력하면 state 업데이트
/>
```

**value + onChange = 세트!** 둘 다 있어야 정상 동작.

왜 이렇게 하나?
→ React가 "진실의 원천(single source of truth)"을 **state 하나**로 관리하기 위해.
→ input에 뭐가 들어있는지 항상 state로 알 수 있음.

### 5. 객체/배열 state 불변 업데이트 (시험 단골!)

```tsx
// === 객체 ===
const [form, setForm] = useState({ name: "", email: "" });

// 올바른 방법: spread로 복사 후 덮어쓰기
setForm({ ...form, name: "새이름" });

// 절대 안 됨: 직접 수정
form.name = "새이름";  // React가 변경 감지 못함 → 화면 안 바뀜!


// === 배열 ===
const [todos, setTodos] = useState<string[]>([]);

// 추가
setTodos([...todos, "새할일"]);

// 삭제 (filter로 해당 index 제외)
setTodos(todos.filter((_, i) => i !== indexToRemove));

// 수정 (map으로 해당 index만 변경)
setTodos(todos.map((todo, i) => i === targetIndex ? "수정됨" : todo));
```

**왜 직접 수정하면 안 되나?**
> React는 state가 **이전과 다른 객체(참조)**인지로 변경을 판단함.
> `form.name = "..."` → 같은 객체니까 "변경 없음" → 렌더링 안 함!
> `{ ...form, name: "..." }` → 새 객체니까 "변경됨!" → 렌더링 함!

---

## 합격 기준

- useState로 카운터, 입력 폼을 만들 수 있으면 OK
- controlled input (value + onChange) 패턴을 코드로 쓸 수 있으면 OK
- "왜 state를 직접 수정하면 안 되나요?" → "참조가 같아서 React가 변경 감지 못함" 이라고 답하면 OK
- e.preventDefault()가 뭐 하는 건지 설명할 수 있으면 OK
- 이벤트 타입 `React.ChangeEvent<HTMLInputElement>` 를 기억하면 OK
