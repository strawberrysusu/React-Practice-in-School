# 1단계: React 시작

> 컴포넌트가 뭔지, TSX 규칙이 뭔지, 화면을 어떻게 조립하는지

---

## 반드시 알아야 하는 것

### 1. 프로젝트 생성
```bash
npm create vite@latest my-app -- --template react-ts
cd my-app
npm install
npm run dev
```
시험에선 "Vite + React + TypeScript 조합"이라는 것만 알면 됨.

### 2. TSX 규칙 (시험 단골!)

| 규칙 | 설명 | 예시 |
|------|------|------|
| 최상위 태그 1개 | 여러 태그를 하나로 감싸야 함 | `<div>...</div>` 또는 `<>...</>` |
| className | HTML의 class 대신 className 사용 | `<div className="box">` |
| 셀프 클로징 | 빈 태그는 반드시 닫기 | `<img />`, `<br />`, `<input />` |
| {} 표현식 | JS 값을 넣을 때 중괄호 사용 | `<p>{user.name}</p>` |

```tsx
// 안 됨: 최상위 태그가 2개
return (
  <h1>제목</h1>
  <p>본문</p>
)

// 됨: Fragment(<>)로 감쌈
return (
  <>
    <h1>제목</h1>
    <p>본문</p>
  </>
)
```

### 3. 컴포넌트 = 대문자로 시작하는 함수

```tsx
// Header.tsx
function Header() {
  return <h1>안녕하세요</h1>;
}
export default Header;

// App.tsx에서 사용
import Header from "./Header";

function App() {
  return (
    <div>
      <Header />  {/* 대문자! 소문자면 HTML 태그로 인식 */}
    </div>
  );
}
```

### 4. 컴포넌트 분리 = 레고 조립
큰 화면을 작은 컴포넌트로 쪼개서 조립하는 게 React의 핵심 아이디어.

```
App
├── Header
├── Main
│   ├── Card
│   ├── Card
│   └── Card
└── Footer
```

---

## 합격 기준

- TSX 규칙 4가지를 말할 수 있으면 OK
- 컴포넌트를 만들고 다른 파일에서 import해서 쓸 수 있으면 OK
- "컴포넌트 이름은 왜 대문자?" → "소문자면 HTML 태그로 인식하니까" 라고 답할 수 있으면 OK
