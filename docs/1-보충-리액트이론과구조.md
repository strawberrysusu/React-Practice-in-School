# 1단계 보충: 리액트 핵심 이론 + 프로젝트 구조

> 교수님 PDF에 나오는데 체크리스트에 빠져있던 내용. 시험에 이론으로 나올 가능성 높음!

---

## 1. 리액트 핵심 철학 4가지 (시험 단골!)

### (1) 컴포넌트 기반 아키텍처
- UI를 작고 재사용 가능한 **컴포넌트**로 나누어 개발
- 레고 블록처럼 조립해서 화면을 만듦

### (2) 가상 DOM (Virtual DOM)
- React는 화면을 바꿀 때 **실제 DOM을 직접 건드리지 않음**
- 대신 가상 DOM(메모리 상의 가짜 DOM)에 변경사항을 먼저 반영
- 그 다음 실제 DOM과 비교해서 **바뀐 부분만 최소한으로 업데이트**
- 왜? → 실제 DOM 조작은 느리니까. 가상 DOM으로 비교 후 필요한 것만 바꾸면 빠름

```
상태 변경 → 가상 DOM에 새 UI 생성 → 이전 가상 DOM과 비교(diffing) → 차이점만 실제 DOM에 반영
```

### (3) 선언적 프로그래밍
- **"어떻게(How)" 가 아니라 "무엇을(What)"** 표현
- "이 상태일 때 화면은 이렇게 보여야 한다"고 선언하면, React가 알아서 DOM을 업데이트
- 개발자가 직접 DOM을 조작할 필요 없음

```tsx
// 선언적 (React 방식) - "무엇을" 보여줄지만 선언
function App() {
  return isLoggedIn ? <Dashboard /> : <LoginPage />;
}

// 명령적 (옛날 방식) - "어떻게" 바꿀지 하나하나 지시
// document.getElementById("root").innerHTML = "..."
// document.createElement("div")...
```

### (4) 단방향 데이터 흐름
- 데이터가 **부모 → 자식** 방향으로만 흐름 (props)
- 자식은 부모의 상태를 **직접 변경할 수 없음**
- 왜? → 데이터 흐름을 예측 가능하게 만들어서 버그를 줄이려고

```
App (부모)
 ├── Header  ← props로 데이터 받음 (위에서 아래로)
 ├── Main    ← props로 데이터 받음
 └── Footer  ← props로 데이터 받음
```

## 2. 프로젝트 실행 흐름 (시험 단골!)

```
npm run dev 실행
    ↓
Vite 개발 서버가 index.html을 로드
    ↓
index.html 안의 <script src="/src/main.tsx"> 로 main.tsx 실행
    ↓
main.tsx에서 createRoot()로 root div를 찾고, <App /> 렌더링
    ↓
App 컴포넌트가 화면에 표시됨
```

### index.html (시작점)
```html
<body>
  <div id="root"></div>  <!-- React 앱이 들어갈 컨테이너 -->
  <script type="module" src="/src/main.tsx"></script>  <!-- 여기서 React 시작 -->
</body>
```
- `<div id="root">` = React 앱이 렌더링되는 위치
- `type="module"` = import/export 사용 가능하게 브라우저에 알려줌

### main.tsx (엔트리 포인트)
```tsx
import { StrictMode } from 'react'
import { createRoot } from 'react-dom/client'
import App from './App.tsx'

createRoot(document.getElementById('root')!).render(
  <StrictMode>
    <App />
  </StrictMode>,
)
```

- **`createRoot()`**: React 18에서 도입된 렌더링 방식. root div를 찾아서 React 앱을 연결
- **`!` (Non-null Assertion)**: TypeScript에게 "이 값은 null이 아니다"라고 보장
- **`StrictMode`**: 개발 환경에서만 동작. 잠재적 버그를 검사해주는 도구. 배포에서는 영향 없음
- **`<App />`**: 최상위(루트) 컴포넌트

### 옛날 방식 vs 새 방식
```tsx
// React 17 (옛날)
ReactDOM.render(<App />, document.getElementById('root'))

// React 18 (현재) - Concurrent Rendering 지원
createRoot(document.getElementById('root')!).render(<App />)
```

## 3. package.json 구조

```json
{
  "name": "my-app",
  "private": true,
  "version": "0.0.0",
  "scripts": {
    "dev": "vite",
    "build": "vite build",
    "preview": "vite preview"
  },
  "dependencies": {
    "react": "^19.0.0",
    "react-dom": "^19.0.0"
  },
  "devDependencies": {
    "typescript": "~5.7.2",
    "vite": "^6.3.1",
    "@types/react": "^19.0.10"
  }
}
```

### dependencies vs devDependencies (시험에 나올 수 있음!)

| | dependencies | devDependencies |
|---|---|---|
| **언제 필요?** | 앱 실행할 때 (배포 포함) | 개발할 때만 |
| **설치 명령** | `npm install 패키지명` | `npm install -D 패키지명` |
| **예시** | react, react-dom | typescript, vite, eslint |

### 패키지 설치 명령어
```bash
npm install axios           # dependencies에 추가 (--save 기본)
npm install -D typescript   # devDependencies에 추가 (--save-dev)
```

## 4. JSX 핵심 정리

### JSX = JavaScript XML
- JavaScript 코드 안에서 HTML 비슷한 문법으로 UI를 정의
- **표준 JavaScript 문법이 아님!** React가 내부적으로 `createElement()`로 변환해서 실행

```tsx
// 우리가 쓰는 JSX
<h1 className="title">안녕</h1>

// React가 내부적으로 변환하는 코드
React.createElement("h1", { className: "title" }, "안녕")
```

### 인라인 스타일 = 객체로 작성 (시험 주의!)
```tsx
// HTML 방식 (JSX에서 안 됨)
<h1 style="color: red; background-color: lightgray">제목</h1>

// JSX 방식 (객체로, 카멜케이스로!)
<h1 style={{ color: "red", backgroundColor: "lightgray" }}>제목</h1>
//          ^^ 첫번째 {} = JSX 표현식
//             ^^ 두번째 {} = JavaScript 객체
```

**CSS 속성 카멜케이스 변환:**
| CSS (하이픈) | JSX (카멜케이스) |
|---|---|
| `background-color` | `backgroundColor` |
| `font-size` | `fontSize` |
| `margin-top` | `marginTop` |
| `text-align` | `textAlign` |

### 표현식 vs 실행문 (JSX 안에서 뭘 쓸 수 있나?)

```tsx
// 표현식 = 값을 반환하는 것 → JSX 안에서 사용 가능
{1 + 2}
{user.name}
{isOk ? "예" : "아니오"}
{items.map(...)}

// 실행문 = 값을 반환하지 않는 것 → JSX 안에서 사용 불가!
{if (isOk) { ... }}       // if문 불가!
{for (let i...) { ... }}  // for문 불가!
{switch (x) { ... }}      // switch문 불가!
```

### JSX 주석
```tsx
return (
  <div>
    {/* JSX 안에서의 주석은 이렇게 */}
    <p>내용</p>
  </div>
)
```

## 5. export vs export default

```tsx
// === export default (파일당 1개, 초보자 권장) ===
// Header.tsx
export default function Header() { ... }

// 사용할 때: 중괄호 없이, 이름 바꿔도 됨
import Header from "./Header"
import MyHeader from "./Header"  // 이름 변경 OK


// === export (여러 개 가능) ===
// utils.tsx
export function add() { ... }
export function subtract() { ... }

// 사용할 때: 반드시 중괄호, 이름 그대로
import { add, subtract } from "./utils"
```

| | export default | export |
|---|---|---|
| 파일당 개수 | 1개만 | 여러 개 가능 |
| import 방식 | 중괄호 없음 | 중괄호 필요 |
| 이름 변경 | 자유 | as 키워드 필요 |

## 6. 클래스 컴포넌트 vs 함수형 컴포넌트

```tsx
// 클래스 컴포넌트 (옛날 방식)
class Hello extends React.Component {
  render() {
    return <h1>안녕</h1>
  }
}

// 함수형 컴포넌트 (현재 방식, 이것만 쓰면 됨)
function Hello() {
  return <h1>안녕</h1>
}
```

| | 클래스 컴포넌트 | 함수형 컴포넌트 |
|---|---|---|
| 문법 | 복잡 (class, render, this) | 간단 (function + return) |
| 상태 관리 | this.state, this.setState | useState Hook |
| 생명주기 | componentDidMount 등 | useEffect Hook |
| 현재 사용 | 거의 안 씀 (레거시) | **React 16.8 이후 표준** |

> 시험 팁: "React 16.8에서 Hook이 도입되면서 함수형 컴포넌트에서도 상태 관리가 가능해졌다" 이 한 줄이면 됨

## 합격 기준

- "가상 DOM이 뭐냐?" → "메모리 상의 가짜 DOM에서 비교 후 바뀐 부분만 실제 DOM에 반영" OK
- "React 실행 흐름을 설명하라" → "index.html → main.tsx → App.tsx" OK
- "dependencies와 devDependencies 차이?" → "배포에도 필요 vs 개발할 때만" OK
- "JSX 안에 if문 쓸 수 있나?" → "없다, 삼항연산자나 &&를 대신 쓴다" OK
